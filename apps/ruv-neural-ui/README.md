# rUv Neural UI — Ruflo Web Console

A **static, local-first** web console for the Ruflo closed-loop sensory
neuromodulation platform (see [ADR-0014](../../docs/adr/0014-web-console.md)).
It demonstrates the whole loop — *identify a target state → deliver a verified
40 Hz stimulus → measure the response → stop safely outside the envelope* — and
**verifies the evidence entirely in your browser**. No backend, no accounts, no
health data leaves the page.

> **Not a medical device.** Research-grade wellness / cognitive-state console on
> safe external sensory channels only.

## Modes (v1)

- **Demo** — deterministic, browser-only. Plays back real evidence bundles
  generated by `ruv-neural neuromod --bundle … --sign` (relaxed converge,
  relaxed fail-safe stop, focused, multimodal gamma).
- **Replay** — import any Ruflo evidence bundle (JSON) and verify it locally:
  schema validity, the recomputed hash chain, receipt integrity + frequency
  tolerance, fail-safe-stop semantics, the Ed25519 signature, and the acceptance
  result.

Real-mode hardware bridges and the guided research workflow are feature-gated
for later phases.

## Develop

```bash
npm install
npm run dev        # local dev server
npm run typecheck  # tsc --noEmit
npm run test       # vitest — schema + verifier + tamper-detection
npm run build      # static build into dist/
npm run preview    # serve the production build on :4173
npm run e2e        # playwright — demo flow + screenshots (needs a browser)
```

## How verification stays honest

The Rust exporter and this TypeScript verifier hash each step from the **same
fixed-precision canonical string**, so the browser **recomputes** the chain
rather than trusting the bundle's self-reported hashes. The Vitest suite proves
parity: `sha256Hex(canonicalPayload(step)) === step.payloadSha256` for real
Rust-generated fixtures.

## Layout

```
src/
  schemas/     Zod schemas for the ruflo-evidence/1 bundle
  verifier/    local hash-chain / receipt / signature / acceptance verification
  fixtures/    bundled, immutable demo evidence (real, signed)
  store/       Zustand store + playback
  components/   charts, cards, playback controls
  screens/     Overview, Live Session, Stimulus Verifier, Biosense,
               ruVector, Safety Envelope, Audit Trail, Witness
tests/         Playwright E2E (also captures README screenshots)
```

## Deployment

Pushes to `main` that touch `apps/ruv-neural-ui/**` build and publish to GitHub
Pages via `.github/workflows/pages.yml`. The Vite `base` is relative, so the
build works at a domain root or a project subpath.
